## Usage Guidelines

### Adding New Terms

1. **Term**: Use UPPERCASE for acronyms, Title Case for phrases
2. **Definition**: Clear, concise explanation (avoid circular definitions)
3. **Context**: Business context, technical context, or regulatory context
4. **Example**: Real usage in requirements or code

### Term Categories

**Regulatory**: Compliance-related terms (GDPR, HIPAA, PCI-DSS)
**Technical**: Architecture, APIs, protocols (REST, OAuth, JWT)
**Business**: Domain-specific jargon (MRR, CAC, LTV, churn)
**Process**: Workflow terms (KYC, onboarding, reconciliation)

### Evidence Standards

- Tier: MEDIUM (community consensus) to AUTHORITATIVE (official standards)
- Source: Link to official definition if available
- Freshness: Review annually for regulatory terms, bi-annually for technical terms

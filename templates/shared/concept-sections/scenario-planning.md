# Scenario Planning Framework (Shell/McKinsey Method)

> **Purpose**: Develop strategic resilience by exploring plausible futures, identifying robust strategies that succeed across multiple scenarios, and establishing early warning systems for strategic pivots.

## Scenario Planning Overview

```
                           UNCERTAINTY AXIS 1
                    [Dimension A: e.g., Market Growth]

                      Low ◄─────────────────► High
                       │                       │
                ┌──────┴──────┐         ┌──────┴──────┐
           High │  SCENARIO 2 │         │  SCENARIO 1 │
                │  [Name]     │         │  [Name]     │
  UNCERTAINTY   │  "Cautious" │         │  "Boom"     │
  AXIS 2        └─────────────┘         └─────────────┘
  [Dimension B:       │                       │
   e.g., Regulation]  │                       │
                ┌──────┴──────┐         ┌──────┴──────┐
            Low │  SCENARIO 4 │         │  SCENARIO 3 │
                │  [Name]     │         │  [Name]     │
                │  "Stagnant" │         │  "Disrupt"  │
                └─────────────┘         └─────────────┘
```

## Uncertainty Axes Selection

### Axis Selection Criteria

| Criterion | Description | Weight |
|-----------|-------------|:------:|
| **Impact** | How much would this uncertainty affect our strategy? | High |
| **Uncertainty** | How unpredictable is this factor (truly uncertain, not just unknown)? | High |
| **Independence** | Are the two axes largely independent of each other? | Medium |
| **Actionability** | Can we take meaningfully different actions based on outcomes? | Medium |

## Scenario Review Cadence

| Review Type | Frequency | Participants | Outputs |
|-------------|:---------:|--------------|---------|
| Indicator Check | Monthly | Strategy Lead | Dashboard update |
| Scenario Assessment | Quarterly | Leadership Team | Probability updates |

| Triggered Review | As needed | Executive Team | Strategy pivot decision |

---

## Scenario Planning Quality Checklist

- [ ] Two independent uncertainty axes selected with clear rationale
- [ ] Four scenarios fully described with distinct narratives
- [ ] Scenario probabilities assigned (must sum to 100%)
- [ ] Each scenario includes market, competitive, customer, tech, and regulatory dimensions
- [ ] Optimal strategy defined for each individual scenario
- [ ] Robustness matrix completed for candidate strategies
- [ ] At least 2 robust strategies identified (work across 3+ scenarios)
- [ ] No-regret moves identified for immediate action
- [ ] Early warning indicators defined for each scenario (minimum 2 per scenario)
- [ ] Monitoring frequency and data sources specified
- [ ] Response playbooks drafted for each scenario
- [ ] Review cadence established

## Integration Notes

- **Feeds into**: Risk matrix (scenario-specific risks), Three horizons (H3 exploration priorities), Financial sensitivity (scenario-based NPV)
- **Depends on**: Market framework (market uncertainties), Competitive analysis (competitor moves), Technology assessment (disruption potential)
- **CQS Impact**: Improves Risk (+4 pts) and Features (+2 pts) scores through future-proofed strategic planning
